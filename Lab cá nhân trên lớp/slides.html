User Feedback Explicit

Phản hồi tường minh từ người dùng

Đánh giá nhanh

  • Binary: 👍 / 👎 dưới mỗi câu trả lời Chat/Composer
  • Single-action: nút Regenerate khi kết quả chưa đạt

Lựa chọn & gợi ý

  • Chọn model, mode Agent / Ask / Manual
  • Nút Apply, Add to Composer
  • Câu hỏi follow-up gợi ý sẵn

Làm rõ & phản hồi sâu

  • Accept / Reject diff — Ctrl+K
  • Sửa lại prompt sau khi kết quả sai
  • Agent hỏi lại khi yêu cầu chưa rõ
  • Rules (.cursor/rules) & Memories
- session.login(user)
+ jwt.verify(token)
[Reject] → "Dùng JWT, không session"

User Feedback Implicit

Phản hồi ngầm định từ người dùng

Hành vi Tín hiệu ngầm
Tab chấp nhận autocomplete Gợi ý inline phù hợp → positive signal
Esc / gõ đè autocomplete Gợi ý không phù hợp → negative signal
Copy code từ Chat (không Apply) Code có giá trị, flow Apply chưa tối ưu
Accept rồi sửa tay ngay Đúng hướng, sai convention / style
Accept → chạy test → fail Syntax OK, logic sai — tín hiệu mạnh nhất
Bỏ qua kết quả, prompt mới Câu trả lời trước không hữu ích

System Feedback

Phản hồi từ hệ thống

💬
Explicit — Chủ động hiển thị
  • Progress: "Thinking...", "Searching codebase...", "Reading 4 files..."
  • @context: hiển thị file nào được dùng làm nguồn

✍️ Generating fix...
🎨
Implicit — Qua thiết kế UI
  • Streaming nhanh → AI viết real-time
  • Delay lâu trước stream → task phức tạp
  • Inline edit che mờ → phạm vi chỉ vùng chọn
  • Chat vs Composer layout khác → mental model khác
  • Autocomplete mờ + Tab → gợi ý nhẹ, user kiểm soát

3 nguyên tắc thu thập feedback hiệu quả

"Thu thập feedback đúng cách → AI học nhanh hơn, user cảm thấy được lắng nghe"

01
Make it easy
✅ Thumbs inline, Accept/Reject ngay trên diff
❌ Thumbs down không mở hộp "vì sao?"
02
Tạo động lực cho user
✅ Rules/Memories giúp AI nhớ preference
❌ Không giải thích feedback cải thiện gì
03
Giải thích cách dùng feedback
✅ @context minh bạch nguồn dữ liệu
❌ Không nói feedback có train model không
❌ Don't
[👍] [👎]
→ User không biết bấm để làm gì
✅ Do
[👍 Hữu ích] [👎 Sai logic]
"Cải thiện gợi ý cho project này"

Tổng kết & đề xuất cải tiến

✅ Điểm mạnh

  • Explicit System Feedback xuất sắc (@context, progress, diff)
  • Accept/Reject kiểm soát chặt trước khi ghi code
  • Implicit loop phong phú: autocomplete, copy, sửa tay, test fail

⚠️ Cần cải thiện

  • Nối Implicit → Explicit khi test fail
  • Thumbs Down + tag: "Sai logic", "Sai convention"
  • Transparency: feedback dùng để làm gì?
  • Motivation copy cho developer
Trade-off: Cursor ưu tiên kiểm soát & an toàn hơn thu feedback nhanh — hợp lý với developer vì sai code tốn kém hơn sai gợi ý. Cải tiến lớn nhất: đóng vòng Implicit → Explicit.

Kết luận

Cursor feedback mạnh ở kiểm soát (Accept/Reject, @context, diff preview) nhưng yếu ở động lực & minh bạch (thumbs up/down).
User Explicit → 👍/👎 · Accept/Reject · Rules
User Implicit → Tab/Esc · Copy · Sửa tay
System Explicit → Progress · @context · Diff
System Implicit → Streaming · UI layout
